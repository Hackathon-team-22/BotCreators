Q2. В задании упоминаются форматы выгрузки HTML и JSON. Надеюсь, мы можем считать, что возможно использовать ТОЛЬКО json выгрузки, учитывая, что пользователь будет специально выгружать чат для использования в боте (и соответственно может выбрать json формат для этого)? Дело в том, что HTML не только гораздо неудобнее парсить, но в нем также отсутствует поле from_id, по которому можно было бы удобно делать дедупликацию. Здесь Json выигрывает по всем параметрам (HTML никакой дополнительной информации не привносит, наоборот, в нем отсутствует важное поле для идентификации пользователя).
Q3. Хочется еще раз убедиться в понимании и принятии фактических ограничений выгрузки, а именно:
 - Поле username отсутствует в выгрузке. Для сообщений в чате мы сможем определять только from_id, которые не ссылаются конкретно на аккаунт Telegram, но которые хотя бы уникальны для каждого пользователя в выгрузке. Важно понимать, что это не username, а внутренний id пользователя в Telegram. Способ достать по нему username отсутствует (по крайней мере в области легального взаимодействия с Telegram из которой выходить не хочется). PS. Однако, в случае упоминаний других пользователей (когда упоминается их username) - то Telegram явно помечает это как mention, и для таких случаев (для упоминаний) - мы сможем явно сказать, что есть username упоминания и указать его в ответе.
 - Имя и фамилия представлена в выгрузке в поле from и является тем именем и фамилией, которые указаны у создателя выгрузки. Например, если я назову своего собеседника как-то иначе (в Telegram можно переименовать собеседника для отображения этого имени только у себя) - то в выгрузке будет именно результат переименования. PS. Кроме полей from_id и from больше никакой информации о пользователе, отправившем сообщение в выгрузке нет.
 - Описание (Bio/About) достать невозможно.
 - Дату регистрации достать невозможно.
 - Наличие канала в профиле (как и его название) достать невозможно.
